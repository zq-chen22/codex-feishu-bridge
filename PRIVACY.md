# 隐私说明

飞行桥是自托管软件。项目维护者不运营中转服务器，代码中也没有面向维护者的遥测或用户分析功能。部署者需要自行决定并负责其飞书租户、Codex 账号、主机及处理数据的合法性。

## 数据如何流动

1. 飞书自建应用接收已配对所有者发送的消息和附件。
2. 消息在本机 SQLite 状态库中排队，并交给本机 Codex App Server。
3. Codex 的进度、最终文本及经批准的文件通过飞书 OpenAPI 返回原会话。
4. 可选的每日统计功能只在部署者显式启用后写入部署者指定的飞书表格。

飞书和 Codex/OpenAI 服务仍分别适用其自身的隐私政策、服务条款和账号设置。

## 本地保存内容

- `config.toml`：非秘密配置、应用 ID 和本地目录。
- `secrets.env`：飞书 App Secret，仅允许当前操作系统用户读取。
- `bridge.sqlite`：消息队列、平台标识符、绑定关系、审批和交付状态。
- `inbox/`：从飞书下载的隔离附件。
- `visual-proxies/`：去除元数据并限制分辨率后的图片代理。
- `outbox/`：Codex 为交付而生成的文件；它们不会被自动当作临时文件删除。

默认情况下，已完成消息正文和已解决的操作记录保存 30 天，服务运行时每天清理一次。可通过 `data_retention_days` 调整；设置为 `0` 会关闭自动数据库清理。待处理、重试中或状态不明确的任务不会自动删除。

## 最小化措施

- 新建飞书群的描述不公开本机绝对路径或原始 Codex thread ID。
- 普通日志不记录原始聊天 ID、用户 ID 或 thread ID，并过滤常见凭据格式。
- 图片在交给模型前会重新编码为受限 JPEG，原始元数据不会传入模型。
- 上传结果必须位于允许的工作区，并通过大小、链接、敏感路径和常见密钥模式检查。
- 默认只有完成配对的单一所有者及相同飞书租户可以下达命令。

## 删除数据

先停止服务，再执行：

```bash
./scripts/uninstall-user-service.sh --purge
```

该命令会删除飞行桥的本机配置、凭据文件、状态库、收件箱、图片代理和交付目录。它不会删除飞书云端已经发送的消息，也不会删除 Codex 自身保存的历史；这些内容需要分别在对应产品中处理。

## 部署者责任

如果你为其他自然人或组织部署本软件，应在接入前说明处理目的、数据范围、保存期限和删除方式，并只授予完成任务所必需的飞书权限。不要把真实生产数据上传到公开 Issue、PR、测试夹具或演示素材中。
