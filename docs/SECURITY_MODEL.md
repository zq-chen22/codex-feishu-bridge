# 安全模型

## 保护目标

- 飞书 App Secret、Codex 凭据和本机其他秘密不离开受控主机。
- 只有完成配对的所有者可以远程下达命令或批准操作。
- 一个飞书会话不能串到另一个 Codex thread。
- 附件和生成文件不能借助路径、软链接、硬链接或替换竞态越过允许目录。
- 服务重启和网络重试不能静默重复具有副作用的 Agent 任务。

## 信任边界

飞行桥信任当前操作系统用户、当前 Codex CLI 安装、部署者创建的飞书应用和已完成配对的所有者账号。飞书平台、网络、第三方 Python 包、聊天文本、附件内容、Agent 输出及工作区代码都不被视为天然可信。

如果攻击者获得操作系统账号、App Secret、有效配对码或已配对飞书账号，项目无法继续维持该边界。应立即停服、轮换凭据、清除配对状态并检查主机。

## 默认防护

- 16 字符随机配对码只保存哈希并在短时间后失效。
- 所有者同时由 app-scoped open ID、租户及可用的 union/user ID 锚定。
- 默认权限为按需审批和工作区写入；远程完全访问需要本机显式开关。
- 状态目录、数据库、配置和 Secret 文件使用当前用户私有权限。
- 飞书群为私有群，群描述隐藏路径和原始 thread ID。
- 图片物理重编码，禁止代理失败时回退到原图。
- 自动上传仅接受专用 outbox 内、大小受限、非链接且未命中常见密钥模式的文件。
- 日志使用哈希引用并脱敏常见凭据与个人标识。

## 未解决风险

- Agent 仍可能被聊天或文件中的提示注入误导；沙箱和人工审批是主要缓解措施。
- 允许联网后，Agent 或工作区程序可能向外部服务发送数据。
- 飞书和 Codex 的上游行为、权限模型或 API 可能变化。
- 密钥模式扫描是防呆措施，不是完整的数据防泄漏系统。
- `danger-full-access` 会显著削弱项目边界，不能通过免责声明变得安全。

部署者应把飞行桥运行在独立的低权限账号或可重建主机中，并定期更新 Codex CLI、Python 依赖和操作系统。
